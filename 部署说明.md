# 部署说明

## 1. 推荐部署方式

推荐：

```text
Coolify + Nixpacks + PostgreSQL
```

项目已经包含：

```text
nixpacks.toml
Dockerfile
docker-compose.yml
scripts/coolify-start.mjs
```

## 2. Coolify 基本设置

Build Pack：

```text
Nixpacks
```

Ports Exposes：

```text
3000
```

Health Check Path：

```text
/api/version
```

Build Command：

```text
npm run prisma:generate && npm run build
```

Start Command：

```text
npm run start:coolify
```

## 3. 必填环境变量

在 Coolify 的网站应用里填写，不是填在 PostgreSQL 服务里。

```env
DATABASE_URL=PostgreSQL 的 Internal URL
NEXT_PUBLIC_APP_URL=https://你的域名
AUTH_SECRET=至少32位随机字符串
SETTINGS_SECRET=另一串至少32位随机字符串
ADMIN_EMAIL=你的管理员邮箱
ADMIN_PASSWORD=你的管理员密码
COOLIFY_INIT_ON_START=true
RESET_ADMIN_PASSWORD_ON_INIT=false
```

## 4. VMOSCloud 环境变量

```env
VMOS_API_BASE_URL=https://api.vmoscloud.com
VMOS_H5_BASE_URL=https://api.vmoscloud.com
VMOS_ACCESS_KEY=你的 AK
VMOS_SECRET_KEY=你的 SK
VMOS_TOKEN_PATH=/vcpcloud/api/padApi/stsTokenByPadCode
ALLOW_VMOS_MOCK=false
```

也可以部署后在管理员后台填写：

```text
/zh/admin/settings/vmos
```

## 5. 数据库初始化

`COOLIFY_INIT_ON_START=true` 时，应用启动会自动尝试：

```text
prisma migrate deploy
prisma/init-production.ts
```

会自动创建：

- 默认套餐
- 管理员账号
- VMOS 设置记录

## 6. 部署后检查

先打开：

```text
https://你的域名/api/version
```

应该返回：

```json
{
  "status": "ok"
}
```

再打开：

```text
https://你的域名/api/health
```

如果数据库正常，应该看到：

```json
{
  "database": "ok"
}
```

如果看到：

```json
{
  "database": "missing_url"
}
```

说明网站应用没有读到 `DATABASE_URL`。

## 7. Bad Gateway 排查

Coolify 显示 Bad Gateway 时，优先检查：

1. `Ports Exposes` 是否是 `3000`
2. Start Command 是否是 `npm run start:coolify`
3. 日志里有没有 `DATABASE_URL is missing`
4. 日志里有没有 Prisma 数据库连接错误
5. `/api/version` 是否能打开
6. PostgreSQL 是否已启动
7. `DATABASE_URL` 是否使用 Internal URL

常见问题：

```text
填了数据库，但填在 PostgreSQL 服务里，而不是网站应用里。
```

正确位置：

```text
Coolify -> Application -> Environment Variables
```

## 8. 本地演示模式

不需要数据库，只看页面：

```bash
npm run preview:demo
```

默认地址：

```text
http://127.0.0.1:4173/zh
```

这个模式只是演示 UI，不代表真实注册登录。
