# 后台和供应商对接说明

## 1. 后台在哪里

中文后台入口：

```text
/zh/admin
```

日语后台入口：

```text
/ja/admin
```

英文后台入口：

```text
/en/admin
```

你需要用管理员账号登录后才能进入。普通客户只能进入 `/zh/app`、`/ja/app`、`/en/app` 个人中心。

## 2. 怎么链接上游云手机供应商

进入：

```text
/zh/admin/settings/vmos
```

这里是运营者专用的供应商 API 设置页。你需要填写：

```text
API Base URL
网页连接 baseUrl
AccessKey / AK
SecretKey / SK
Token 请求路径
默认清晰度
默认码率
默认帧率
```

这些配置只保存在服务器端，客户浏览器不会拿到 AK/SK。

## 3. 怎么录入云手机

先在上游供应商后台购买云手机，拿到每台手机的设备编号后进入：

```text
/zh/admin/phones
```

录入：

```text
内部编号：JP-001
供应商设备编号：PAD-JP-001
地区：日本
供应商到期时间：2026-06-10
备注：30日标准机
```

客户前台不会看到“供应商设备编号”，只会看到“云手机”和到期时间。

## 4. 怎么分配给客户

客户付款后进入：

```text
/zh/admin/assignments
```

选择：

```text
客户邮箱
空闲供应商设备
已付款订单
客户到期时间
```

保存后，客户登录个人中心 `/zh/app` 就能看到自己的云手机。

## 5. 为什么客户不知道供应商

客户打开云手机时，流程是：

```text
客户浏览器
↓
你的服务器接口
↓
服务器检查登录、订单、归属、到期、状态
↓
服务器使用 AK/SK 请求上游供应商
↓
服务器返回临时网页连接授权
↓
客户看到欧阳日本事业部的云手机画面
```

前台和个人中心只显示：

```text
云手机
打开云手机
网页远程连接
临时授权
到期时间
续费
联系客服
```

不会显示：

```text
供应商名称
AK/SK
供应商设备编号
上游 API 地址
```
