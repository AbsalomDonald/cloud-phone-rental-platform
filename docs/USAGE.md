# 使用说明

## 1. 客户看到什么

客户打开前台：

```text
/zh
/ja
/en
```

前台只做三件事：

- 介绍欧阳日本事业部的云手机服务
- 展示套餐和使用流程
- 引导客户注册 / 登录

## 2. 客户注册登录后

客户进入：

```text
/zh/app
```

个人中心包含：

```text
个人中心概览
我的云手机
订单记录
续费
客服工单
```

### 我的云手机

路径：

```text
/zh/app/phones
```

客户能看到：

- 云手机名称
- 状态
- 到期时间
- 打开云手机按钮

客户点击“打开云手机”时，后端接口会执行：

```text
客户点击打开
↓
你的服务器检查登录状态
↓
检查这台云手机是否属于该客户
↓
检查订单是否已付款
↓
检查是否过期或禁用
↓
向上游供应商申请临时网页连接授权
↓
前端加载云手机网页画面
↓
客户看到云手机画面
```

### 订单记录

路径：

```text
/zh/app/orders
```

客户能查看：

- 订单编号
- 套餐
- 金额
- 状态
- 时间

### 续费 / 下单

路径：

```text
/zh/app/renew
```

客户选择套餐后，系统会创建一条 `pending` 订单。

第一版建议使用：

```text
7日体验：一次性付款
30日标准：一次性付款
商务批量：联系客服报价
```

### 客服工单

路径：

```text
/zh/app/support
```

客户可以提交问题。工单会保存到数据库，运营者在后台回复后，客户个人中心可见。

## 3. 运营者后台怎么用

运营者进入：

```text
/zh/admin
```

后台包含：

```text
后台首页
用户管理
订单管理
套餐管理
云手机库存
分配供应商设备
客服工单
供应商设置
操作日志
```

### 套餐管理

路径：

```text
/zh/admin/plans
```

可以管理：

- 新增套餐
- 编辑套餐
- 上架 / 下架

这些数据保存到 `plans` 表。

### 云手机库存管理

路径：

```text
/zh/admin/phones
```

你在上游云手机供应商后台购买手机后，把资源录入你的后台：

```text
内部编号：JP-001
供应商设备编号：PAD-JP-001
地区：日本
供应商到期时间：2026-06-10
备注：30日标准机
```

这些数据保存到 `phones` 表。

### 分配供应商设备

路径：

```text
/zh/admin/assignments
```

运营流程：

```text
客户付款成功
↓
订单变成已付款
↓
运营者选择一个空闲供应商设备
↓
绑定给客户邮箱
↓
设置客户到期时间
↓
客户个人中心出现云手机
```

绑定关系保存到 `assignments` 表。

### 客服工单

路径：

```text
/zh/admin/support
```

可以查看客户提交的问题、回复、关闭工单。

工单保存到 `support_tickets` 表。

### 供应商 API 设置

路径：

```text
/zh/admin/settings/vmos
```

运营者配置：

- API Base URL
- 网页连接 baseUrl
- AccessKey / AK
- SecretKey / SK
- 默认清晰度
- 默认码率
- 默认帧率
- Token 请求限流策略

重要：AK/SK 只能保存在服务器端，不能给客户浏览器。客户前台和个人中心只显示“云手机”，不会显示供应商名称、AK/SK 或供应商设备编号等内部信息。

### 订单管理

路径：

```text
/zh/admin/orders
```

第一版支持手动运营流程：

```text
客户下单
↓
订单 pending
↓
客户通过你配置的支付方式付款
↓
运营者确认到账
↓
后台把订单改为 paid
↓
进入分配页面绑定供应商设备
```

后续接 Stripe / PayPal 回调后，订单可自动从 `pending` 改为 `paid`。

## 4. 管理员每天操作顺序

```text
1. 登录后台
2. 看订单是否有已付款
3. 进入云手机库存，确认有空闲供应商设备
4. 进入分配页面，把供应商设备绑定给客户
5. 客户收到通知后进入个人中心使用
6. 查看客服工单，回复问题
7. 定期检查即将到期客户，引导续费
```

## 5. 当前 MVP 限制

当前版本已经具备数据库和权限基础，但还有这些运营限制：

- 支付是手动确认，自动支付回调需接 Stripe / PayPal
- 邮件通知未接入，需要配置 SMTP / Resend / SendGrid
- 云手机网页画面初始化需要按你拿到的供应商 SDK 文档补最后一段初始化代码
- 法律页面需要你提供正式公司信息和条款内容
- 生产环境必须配置数据库备份和 HTTPS

所以它适合做第一版人工运营 MVP：你确认付款、手动分配供应商设备，客户登录后使用。
